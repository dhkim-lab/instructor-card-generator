## 1. 개요

### 1.1 배경
현재 강사 프로필 카드는 수작업으로 제작되며, 강사별 원본 자료(PPTX, DOCX 등)에서 정보를 추출하고 PDF 템플릿에 맞춰 수동 편집하는 과정이 반복된다. 강사 수가 늘어날수록 제작 시간과 품질 편차가 커지는 문제가 있다.

### 1.2 목적
강사 프로필 자료를 업로드하면 AI가 자동으로 정보를 추출하고, 팀제이커브 표준 템플릿에 맞춘 강사 카드를 생성·편집·PDF 추출할 수 있는 사내 웹 서비스를 구축한다.

### 1.3 핵심 가치
- **시간 절약**: 강사 1인당 카드 제작 시간 30분 → 5분 이내
- **품질 일관성**: 표준 템플릿 기반으로 모든 강사 카드의 디자인 통일
- **팀 협업**: 누구나 웹에서 접근하여 강사 카드 생성·수정 가능
- **자동 아카이빙**: 생성된 PDF가 Google Drive에 자동 저장, 링크로 즉시 공유

## 3. 기능 요구사항

### 3.1 파일 업로드 및 AI 데이터 추출

**설명**: 사용자가 강사 프로필 원본 파일을 업로드하면, AI가 자동으로 강사 정보를 구조화된 데이터로 추출한다.

**지원 파일 형식**: PPTX, DOCX, PDF

**추출 데이터 항목**:

| 항목 | 설명 | 필수 여부 |
|------|------|-----------|
| 성명 | 강사 이름 | 필수 |
| 직함/타이틀 | 전문 분야 요약 (영문) | 필수 |
| 프로필 요약 | 1~2문장 소개글 | 필수 |
| 프로필 사진 | 이미지 파일 자동 추출 | 선택 |
| 학력 및 강의 분야 | 학위, 전공, 강의 분야 | 필수 |
| 주요 경력 | 기간, 소속, 직위 (테이블) | 필수 |
| 프로젝트/작품 | 프로젝트명, 소속, 장르 등 | 선택 |
| 전시/심사 이력 | 구분(전시/심사), 내용 | 선택 |
| 기타 사항 | 앰배서더, 산학협력 등 | 선택 |
| 세부 강의 이력 | 카테고리별 강의 목록 | 선택 |

**AI 추출 사양**:
- Claude API 활용 (claude-sonnet-4-6 이상)
- 파일에서 텍스트 추출 후 구조화된 JSON으로 변환
- 추출 결과를 사용자에게 미리보기로 제시하여 확인/수정 가능

**수용 조건**:
- [ ] PPTX, DOCX, PDF 각각 업로드 시 정상적으로 텍스트 추출
- [ ] 추출된 데이터가 편집 가능한 폼으로 표시
- [ ] 이미지(프로필 사진) 자동 추출 및 미리보기
- [ ] 추출 실패 시 수동 입력 폼으로 fallback

### 3.2 강사 카드 미리보기

**설명**: 추출/입력된 데이터를 기반으로 팀제이커브 표준 템플릿에 맞춘 강사 카드를 실시간 미리보기로 보여준다.

**템플릿 사양**:
- A4 세로 (210mm × 297mm)
- 헤더: 왼쪽 "TEAM J-CURVE" (파란색 볼드), 오른쪽 "팀제이커브 강사 프로필"
- 첫 페이지 상단: 파란 그라데이션 바 (28mm 높이) + 흰색 TEAM J-CURVE 로고 중앙
- 프로필 영역: 사진(34×43mm) + 이름, 직함, 소개글
- 섹션: 학력, 주요 경력, 프로젝트, 전시/심사, 기타 사항, 세부 강의 이력
- 푸터: 왼쪽 "(주) 팀제이커브 | CONFIDENTIAL", 오른쪽 페이지 번호
- 색상: Primary #3346FF, Navy #0a0a2e, 테이블 헤더 Navy 배경 + 흰색 텍스트

**수용 조건**:
- [ ] 데이터 입력/수정 시 미리보기가 실시간 반영 (또는 "미리보기 갱신" 버튼)
- [ ] 실제 PDF 출력과 동일한 레이아웃으로 미리보기 표시
- [ ] 페이지가 여러 장일 경우 페이지 넘김 표현
- [ ] 모바일에서도 미리보기 접근 가능 (반응형)

### 3.3 텍스트 편집

**설명**: AI가 추출한 데이터를 사용자가 직접 수정할 수 있는 편집 인터페이스를 제공한다.

**편집 UI 구성**:
- 좌측: 편집 패널 (항목별 입력 폼)
- 우측: 실시간 미리보기 패널
- 항목 추가/삭제: 경력, 프로젝트, 강의 이력 등 행 단위 추가·삭제
- 프로필 사진: 별도 이미지 업로드 또는 교체 가능
- 섹션 표시/숨김: 해당 강사에 해당하지 않는 섹션은 토글로 숨김 처리

**수용 조건**:
- [ ] 모든 추출 항목에 대해 인라인 편집 가능
- [ ] 테이블 형식 항목(경력, 프로젝트 등)은 행 추가/삭제/순서변경
- [ ] 프로필 사진 교체 및 크롭 기능
- [ ] 편집 중 데이터 임시 저장 (브라우저 세션 또는 서버)
- [ ] 섹션별 표시/숨김 토글

### 3.4 PDF 추출 및 Google Drive 자동 저장

**설명**: 최종 편집된 강사 카드를 PDF로 생성하고, Google Drive 지정 폴더에 자동 저장한 뒤 공유 링크를 사용자에게 제공한다.

**PDF 생성 사양**:
- 서버사이드 렌더링 (Playwright 또는 Puppeteer headless browser)
- A4, 300dpi 품질
- NanumGothic 폰트 임베딩
- 헤더/푸터 with 페이지 번호

**Google Drive 연동**:
- Google Drive API v3 사용
- 저장 경로: 팀 드라이브 내 기존 강사카드 폴더 (폴더 ID 설정으로 관리)
- 파일명 규칙: `강사카드_{강사명}_{YYYYMMDD}.pdf`
- 저장 완료 후 Google Drive 공유 링크 자동 생성 → 사용자에게 표시
- 동일 강사 카드 업데이트 시 버전 관리 (기존 파일 유지 + 신규 파일 추가)

**수용 조건**:
- [ ] "PDF 생성" 버튼 클릭 시 PDF 다운로드 + Drive 자동 저장
- [ ] 저장 완료 후 Google Drive 링크가 화면에 표시
- [ ] 링크 클릭 시 Drive에서 바로 열람 가능
- [ ] 동일 강사의 기존 카드가 있어도 덮어쓰지 않음 (날짜 기반 버전)

## 5. 기술 아키텍처

### 5.1 기술 스택

| 계층 | 기술 | 선정 이유 |
|------|------|-----------|
| Frontend | Next.js (React) + TypeScript | Vercel 최적화, SSR/SSG, 빠른 개발 |
| UI 라이브러리 | Tailwind CSS + shadcn/ui | 일관된 디자인 시스템, 빠른 UI 구축 |
| Backend | Next.js API Routes (Serverless) | 별도 서버 불필요, Vercel에서 바로 배포 |
| AI 추출 | Anthropic Claude API | 비정형 문서에서 구조화 데이터 추출 |
| PDF 생성 | Vercel Serverless + Playwright | 서버사이드 headless PDF 렌더링 |
| 인증 | NextAuth.js + Google OAuth | Google Workspace 연동, 간편 설정 |
| 스토리지 | Google Drive API v3 | 팀 드라이브 직접 저장, 링크 공유 |
| 배포 | Vercel | 자동 배포, 프리뷰 환경, 무료 티어 활용 |

### 5.2 시스템 흐름

```
[사용자]
  → Google 로그인
  → 파일 업로드 (PPTX/DOCX/PDF)
  → [Serverless Function] 파일 파싱 + Claude API 호출
  → 구조화 JSON 반환
  → [Frontend] 편집 폼 + 실시간 미리보기
  → 사용자 편집·확인
  → "PDF 생성" 클릭
  → [Serverless Function] HTML → Playwright PDF 생성
  → [Google Drive API] 팀 드라이브 폴더에 업로드
  → 공유 링크 반환 → 사용자에게 표시
```

### 5.3 주요 API 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| POST | `/api/upload` | 파일 업로드 및 AI 추출 |
| POST | `/api/preview` | 미리보기 HTML 생성 |
| POST | `/api/generate-pdf` | PDF 생성 + Drive 저장 |
| GET | `/api/templates` | 카드 템플릿 목록 |

## 7. Google Drive 연동 상세

### 7.1 폴더 구조
```
팀 드라이브/
└── 강사카드/               ← 기존 폴더 (설정에서 폴더 ID 지정)
    ├── 강사카드_심원문_20260325.pdf
    ├── 강사카드_김민준_20260320.pdf
    └── ...
```

### 7.2 권한 모델
- 서비스 계정(Service Account) 또는 사용자 OAuth 토큰으로 Drive API 호출
- 팀 드라이브 폴더에 파일 생성 권한 필요
- 생성된 파일은 팀 드라이브 기본 공유 설정 상속
- 사용자에게 반환하는 링크: `https://drive.google.com/file/d/{fileId}/view`

### 7.3 파일 관리 정책
- 동일 강사 이름으로 재생성 시: 날짜 기반으로 신규 파일 생성 (덮어쓰기 금지)
- 파일명 충돌 시: `강사카드_심원문_20260325_v2.pdf` 형태로 버전 부여
- 삭제 정책: 웹 서비스에서 삭제 기능 미제공 (Drive에서 직접 관리)
